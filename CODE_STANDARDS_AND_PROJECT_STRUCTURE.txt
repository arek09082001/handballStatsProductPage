## General Project Structure

### Root Directory Organization
```
projectName/
├── app/                          # Next.js 15 App Router (Pages & Layouts)
│   ├── (auth)/                   # Route groups for authenticated pages
│   ├── api/                      # API routes (Next.js App Router)
│   │   ├── auth/                 # Authentication endpoints
│   │   ├── teams/                # Team management endpoints
│   │   ├── organisations/        # Organisation endpoints
│   │   └── [feature]/            # Feature-specific API routes
│   │       ├── route.ts          # GET, POST, PUT, DELETE handlers
│   │       └── [id]/             # Dynamic route segments
│   │           └── route.ts      # Individual resource operations
│   ├── globals.css               # Global styles
│   ├── layout.tsx                # Root layout
│   └── page.tsx                  # Home page
├── components/                   # Reusable UI components
│   ├── custom-ui/                # Project-specific components
│   ├── forms/                    # Form-related components
│   ├── ui/                       # Base UI components (shadcn/ui)
│   └── wrapper/                  # Provider wrapper components
├── features/                     # Feature-based modules
│   ├── [feature-name]/           # Individual feature modules
│   │   ├── components/           # Feature-specific components
│   │   ├── hooks/                # Feature-specific hooks
│   │   └── pages/                # Feature page components
├── hooks/                        # Global custom hooks
├── lib/                          # Core utilities and configurations
│   ├── api/                      # API layer and interfaces
│   ├── authentication/           # Auth configuration
│   ├── hooks/                    # Global data hooks
│   └── utils/                    # Utility functions
├── messages/                     # Internationalization files
│   ├── en.json                   # English translations
│   └── de.json                   # German translations
└── public/                       # Static assets
```

### Key Architectural Principles
- **Feature-Based Organization**: Each domain (organizations, contacts, teams) has its own feature folder
- **Separation of Concerns**: UI components, business logic, and data fetching are separated
- **Reusability**: Common components and utilities are shared across features
- **Type Safety**: Full TypeScript coverage with strict typing

## Naming Conventions

### File and Folder Naming
- **Folders**: `kebab-case` for all directories
  - ✅ `postal-addresses`, `team-management`, `custom-ui`
  - ❌ `PostalAddresses`, `teamManagement`

- **File Names**: `kebab-case` for all file names
  - ✅ `section-title-description.tsx`, `postal-address-form.tsx`, `use-team-members.tsx`
  - ❌ `SectionTitleDescription.tsx`, `PostalAddressForm.tsx`, `useTeamMembers.tsx`

- **React Components**: Export names use `PascalCase`, but file names use `kebab-case`
  - ✅ `section-title-description.tsx` exports `SectionTitleDescription`
  - ❌ `SectionTitleDescription.tsx` or `sectionTitleDescription` export

- **Hooks**: File names `kebab-case` starting with "use-", export names `camelCase`
  - ✅ `use-grid-layout-store.tsx` exports `useGridLayoutStore`
  - ❌ `UseGridLayout.tsx` or `team-members-hook.tsx`

- **Utilities and Configs**: `kebab-case` for all utility and configuration files
  - ✅ `form-config.tsx`, `error-boundary.tsx`, `auth-config.tsx`
  - ❌ `FormConfig.tsx`, `ErrorBoundary.tsx`, `AuthConfig.tsx`

- **API Interfaces**: `kebab-case` for files, `PascalCase` for interface names
  - ✅ `team-role.tsx` with `interface TeamRole`
  - ❌ `TeamRole.tsx` or `interface teamRole`

- **API Routes**: `kebab-case` for route directories, `route.ts` for handlers
  - ✅ `app/api/team-management/route.ts`, `app/api/postal-addresses/[id]/route.ts`
  - ❌ `app/api/TeamManagement/route.ts`, `app/api/postalAddresses/[id]/route.ts`

### File Extensions
- **React Components, Hooks, and JSX-related files**: Always use `.tsx`
  - ✅ All components, custom hooks, context providers, higher-order components
  - ✅ `section-title-description.tsx`, `use-team-members.tsx`, `error-boundary.tsx`

- **Pure TypeScript files**: Use `.ts` only for files without JSX/React
  - ✅ API routes (`route.ts`), pure utility functions, type-only files
  - ✅ `app/api/teams/route.ts`, `constants.ts`, `types.ts`

- **Configuration files**: Use `.tsx` for most project files to maintain consistency
  - ✅ `form-config.tsx`, `auth-config.tsx`

### Variable and Function Naming
- **Variables**: `camelCase`
  - ✅ `organisationId`, `teamMemberData`, `isLoading`
  - ❌ `organisation_id`, `TeamMemberData`

- **Functions**: `camelCase` with descriptive verbs
  - ✅ `fetchTeamMembers`, `validateFormData`, `handleSubmit`
  - ❌ `getTeamMembers`, `validate`, `submit`

- **Constants**: `UPPER_SNAKE_CASE`
  - ✅ `CONFIGURATION_OBJECTS`, `FILTER_KEY`
  - ❌ `configurationObjects`, `filterKey`

- **API Parameters**: Prefixed with `p` in Hungarian notation
  - ✅ `pId`, `pOrganisationId`, `pTeamMember`
  - ❌ `id`, `organisationId`, `teamMember`

## Component Architecture

### Component Structure Standards
All React components follow these patterns:

```tsx
'use client'; // For client components only

import { useState, useEffect, useCallback } from 'react';
import { useTranslations } from 'next-intl';
import { toast } from 'sonner';

// External library imports
import { Card } from '@/components/ui/card';
import { Button } from '@/components/ui/button';

// Internal component imports
import SectionTitleDescription from '@/components/custom-ui/section-title-description';
import ErrorBoundary from '@/lib/utils/error-boundary';

// Hook imports
import useTeamMembers from '@/lib/hooks/use-team-members';

// Type imports
import { TeamMember } from '@/lib/api/interfaces/team';

interface ComponentProperties {
  organisationId: string;
  onUpdate?: (data: TeamMember) => void;
}

export default function ComponentName({ 
  organisationId,
  onUpdate 
}: ComponentProperties) {
  const t = useTranslations('teamMembers');
  const [isLoading, setIsLoading] = useState(false);
  
  const { teamMembers, isError, refetch } = useTeamMembers(organisationId);

  const handleSubmit = useCallback(async (data: TeamMember) => {
    try {
      setIsLoading(true);
      // Business logic here
      onUpdate?.(data);
      toast.success(t('successMessage'));
    } catch (error) {
      toast.error(t('errorMessage'));
    } finally {
      setIsLoading(false);
    }
  }, [onUpdate, t]);

  if (isError) {
    return <ErrorComponent onRetry={refetch} />;
  }

  return (
    <Card className="p-6">
      <SectionTitleDescription
        title={t('title')}
        description={t('description')}
      />
      {/* Component JSX */}
    </Card>
  );
}
```

### Component Types
1. **Page Components**: Located in `features/[feature]/pages/`
   - Handle route-level logic and layout
   - Combine multiple feature components
   - Example: `PostalAddressesPage`

2. **Feature Components**: Located in `features/[feature]/components/`
   - Domain-specific business logic
   - Reusable within the feature
   - Example: `TeamAssignment`, `PostalAddressForm`

3. **UI Components**: Located in `components/ui/`
   - Pure presentation components
   - No business logic
   - Example: `Button`, `Card`, `Dialog`

4. **Custom UI Components**: Located in `components/custom-ui/`
   - Project-specific reusable components
   - May contain business logic
   - Example: `SectionTitleDescription`, `LoadingButton`

## Data Fetching & State Management

### React Query Pattern
All data fetching uses TanStack Query with consistent patterns:

```tsx
// lib/hooks/use-team-members.tsx
import { useQuery, useMutation, UseQueryResult } from '@tanstack/react-query';
import LiveApi from '@/lib/api/live-api';

const FILTER_KEY = 'team-members';

// Fetch hook pattern
const useTeamMembersGet = (pTeamId?: string): UseQueryResult<TeamMember[], unknown> => {
  const filter = useFilterStore((state) => state.getFilter(FILTER_KEY));
  
  return useQuery<TeamMember[]>({
    queryKey: [FILTER_KEY, pTeamId, filter],
    queryFn: async () => {
      if (!pTeamId) {
        throw new Error('Team ID is required');
      }
      return await LiveApi.getTeamMembers(pTeamId, filter);
    },
    enabled: !!pTeamId,
    refetchOnWindowFocus: false,
    refetchOnReconnect: false,
  });
};

// Mutation hook pattern
const useUpdateTeamMembers = (): UseMutationResult<
  TeamMember[],
  unknown,
  { pTeamId: string; pTeamMembers: TeamMembersBody },
  unknown
> => {
  return useMutation({
    mutationFn: async (variables: {
      pTeamId: string;
      pTeamMembers: TeamMembersBody;
    }) => {
      return await LiveApi.updateTeamMembers(
        variables.pTeamId,
        variables.pTeamMembers,
      );
    },
  });
};

// Export pattern with custom hook wrapper
export const useTeamMembers = (pTeamId?: string) => {
  const {
    data: teamMembers,
    isLoading,
    isError,
    refetch,
  } = useTeamMembersGet(pTeamId);

  const { mutateAsync: updateTeamMembers, isPending: updatePending } = useUpdateTeamMembers();

  return {
    teamMembers,
    isLoading,
    isError,
    refetch,
    updateTeamMembers,
    updatePending,
  };
};
```

### State Management Patterns
1. **Server State**: React Query for API data
2. **Client State**: Zustand for global app state
3. **Form State**: React Hook Form for form management
4. **Component State**: useState for local component state

### Query Key Patterns
- Use consistent naming: `[FILTER_KEY, ...params, filter]`
- Include all dependencies that affect the query
- Use the same key for invalidation

## API Layer

### API Structure
The API layer is centralized in `lib/api/live-api.tsx`:

```tsx
export default class LiveApi extends AzureApi {
  // CRUD pattern for each entity
  public static async getTeams(pFilter?: Record<string, string>): Promise<Team[]> {
    const response = (await this._GET('dynamic/teams', pFilter)) as Team[];
    if (Array.isArray(response)) {
      return defaultResponseParser(response) as Team[];
    }
    throw new Error('Unexpected response Type');
  }

  public static async getTeam(pId: string): Promise<Team> {
    const response = (await this._GET(`dynamic/teams/${pId}`)) as Team[];
    if (Array.isArray(response)) {
      return (defaultResponseParser(response) as Team[])[0];
    }
    throw new Error('Unexpected response Type');
  }

  public static async createTeam(pBody: TeamBody): Promise<Team> {
    const response = (await this._POST(
      'dynamic/teams',
      pBody as unknown as BodyInit,
    )) as Team[];
    if (Array.isArray(response)) {
      return (defaultResponseParser(response) as Team[])[0];
    }
    throw new Error('Unexpected response Type');
  }

  public static async updateTeam(pId: string, pBody: TeamBody): Promise<Team> {
    const response = (await this._PUT(
      `dynamic/teams/${pId}`,
      pBody as unknown as BodyInit,
    )) as Team[];
    if (Array.isArray(response)) {
      return (defaultResponseParser(response) as Team[])[0];
    }
    throw new Error('Unexpected response Type');
  }

  public static async deleteTeam(pId: string): Promise<void> {
    await this._DELETE(`dynamic/teams/${pId}`);
  }
}
```

### API Route Implementation Standards
All API routes follow consistent patterns using Next.js App Router conventions:

```ts
// app/api/teams/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { z } from 'zod';
import { withAuth } from '@/lib/api/middleware/auth';
import { handleApiError } from '@/lib/api/utils/error-handler';
import LiveApi from '@/lib/api/live-api';
import { Team, TeamBody } from '@/lib/api/interfaces/team';

// Request validation schemas
const CreateTeamSchema = z.object({
  Code: z.string().min(1),
  Description: z.string().min(1),
  TeamTypeId: z.string().uuid(),
  Comment: z.string().optional(),
  IsActive: z.boolean().optional().default(true),
});

const GetTeamsQuerySchema = z.object({
  page: z.coerce.number().min(1).optional(),
  limit: z.coerce.number().min(1).max(100).optional(),
  search: z.string().optional(),
  isActive: z.coerce.boolean().optional(),
});

// GET /api/teams - Fetch teams with filtering
export async function GET(request: NextRequest) {
  try {
    const { searchParams } = new URL(request.url);
    const queryParams = Object.fromEntries(searchParams);
    
    // Validate query parameters
    const validatedQuery = GetTeamsQuerySchema.parse(queryParams);
    
    // Build filter object for API call
    const filter: Record<string, string> = {};
    if (validatedQuery.search) filter.search = validatedQuery.search;
    if (validatedQuery.isActive !== undefined) filter.isActive = String(validatedQuery.isActive);
    if (validatedQuery.page) filter.page = String(validatedQuery.page);
    if (validatedQuery.limit) filter.limit = String(validatedQuery.limit);

    const teams = await LiveApi.getTeams(filter);

    return NextResponse.json({
      success: true,
      data: teams,
      meta: {
        page: validatedQuery.page || 1,
        limit: validatedQuery.limit || 20,
      },
    });
  } catch (error) {
    return handleApiError(error);
  }
}

// POST /api/teams - Create new team
export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    const validatedData = CreateTeamSchema.parse(body);

    const teamBody: TeamBody = {
      p_Code: validatedData.Code,
      p_Description: validatedData.Description,
      p_TeamTypeId: validatedData.TeamTypeId,
      p_Comment: validatedData.Comment,
      p_IsActive: validatedData.IsActive,
    };

    const createdTeam = await LiveApi.createTeam(teamBody);

    return NextResponse.json({
      success: true,
      data: createdTeam,
      message: 'Team created successfully',
    }, { status: 201 });
  } catch (error) {
    return handleApiError(error);
  }
}
```

```ts
// app/api/teams/[id]/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { z } from 'zod';
import { handleApiError } from '@/lib/api/utils/error-handler';
import LiveApi from '@/lib/api/live-api';
import { TeamBody } from '@/lib/api/interfaces/team';

const UpdateTeamSchema = z.object({
  Code: z.string().min(1).optional(),
  Description: z.string().min(1).optional(),
  TeamTypeId: z.string().uuid().optional(),
  Comment: z.string().optional(),
  IsActive: z.boolean().optional(),
});

// GET /api/teams/[id] - Fetch single team
export async function GET(
  request: NextRequest,
  { params }: { params: { id: string } }
) {
  try {
    const team = await LiveApi.getTeam(params.id);
    
    return NextResponse.json({
      success: true,
      data: team,
    });
  } catch (error) {
    return handleApiError(error);
  }
}

// PUT /api/teams/[id] - Update team
export async function PUT(
  request: NextRequest,
  { params }: { params: { id: string } }
) {
  try {
    const body = await request.json();
    const validatedData = UpdateTeamSchema.parse(body);

    // Only include provided fields in the update
    const teamBody: Partial<TeamBody> = {};
    if (validatedData.Code) teamBody.p_Code = validatedData.Code;
    if (validatedData.Description) teamBody.p_Description = validatedData.Description;
    if (validatedData.TeamTypeId) teamBody.p_TeamTypeId = validatedData.TeamTypeId;
    if (validatedData.Comment !== undefined) teamBody.p_Comment = validatedData.Comment;
    if (validatedData.IsActive !== undefined) teamBody.p_IsActive = validatedData.IsActive;

    const updatedTeam = await LiveApi.updateTeam(params.id, teamBody as TeamBody);

    return NextResponse.json({
      success: true,
      data: updatedTeam,
      message: 'Team updated successfully',
    });
  } catch (error) {
    return handleApiError(error);
  }
}

// DELETE /api/teams/[id] - Delete team
export async function DELETE(
  request: NextRequest,
  { params }: { params: { id: string } }
) {
  try {
    await LiveApi.deleteTeam(params.id);

    return NextResponse.json({
      success: true,
      message: 'Team deleted successfully',
    });
  } catch (error) {
    return handleApiError(error);
  }
}
```

### API Middleware and Utilities
Create reusable middleware and utilities for common API functionality:

```ts
// lib/api/middleware/auth.tsx
import { NextRequest, NextResponse } from 'next/server';
import { verifyToken } from '@/lib/authentication/token-utils';

export function withAuth(handler: Function) {
  return async (request: NextRequest, ...args: any[]) => {
    try {
      const token = request.headers.get('authorization')?.replace('Bearer ', '');
      
      if (!token) {
        return NextResponse.json(
          { success: false, error: 'Authentication required' },
          { status: 401 }
        );
      }

      const user = await verifyToken(token);
      
      // Add user context to request
      (request as any).user = user;
      
      return handler(request, ...args);
    } catch (error) {
      return NextResponse.json(
        { success: false, error: 'Invalid authentication' },
        { status: 401 }
      );
    }
  };
}
```

```ts
// lib/api/utils/error-handler.tsx
import { NextResponse } from 'next/server';
import { ZodError } from 'zod';

export function handleApiError(error: unknown) {
  console.error('API Error:', error);

  // Zod validation errors
  if (error instanceof ZodError) {
    return NextResponse.json(
      {
        success: false,
        error: 'Validation failed',
        details: error.errors.map(err => ({
          field: err.path.join('.'),
          message: err.message,
        })),
      },
      { status: 400 }
    );
  }

  // Known API errors
  if (error instanceof Error) {
    if (error.message.includes('not found')) {
      return NextResponse.json(
        { success: false, error: 'Resource not found' },
        { status: 404 }
      );
    }
    
    if (error.message.includes('Unauthorized')) {
      return NextResponse.json(
        { success: false, error: 'Unauthorized access' },
        { status: 401 }
      );
    }
  }

  // Generic server error
  return NextResponse.json(
    { success: false, error: 'Internal server error' },
    { status: 500 }
  );
}
```

## TypeScript Standards

### Interface Design Patterns
1. **Entity Interfaces**: Extend `Base` interface
2. **Request Bodies**: Separate interfaces with `p_` prefixed properties
3. **Form Data**: Separate interfaces for form-specific data shapes
4. **Component Props**: Inline interfaces for simple props, separate for complex ones

## Error Handling

### Error Handling Patterns
1. **API Errors**: Caught in mutation hooks and displayed via toast
2. **Component Errors**: Error boundaries for graceful degradation
3. **Form Errors**: Zod validation with translated messages
4. **Network Errors**: Retry mechanisms with user feedback

### Error Display Standards
```tsx
// Toast notifications for user actions
try {
  await createTeam({ pTeam: teamData });
  toast.success(t('successfullCreate', { placeholder: 'Team' }));
} catch (error) {
  toast.error(t('errorMessages.errorOnCreate'));
  throw error;
}

// Error boundaries for component failures
<ErrorBoundary
  fallBackComponent={
    <Button
      variant='destructive'
      className='h-10 w-full'
      onClick={(e) => {
        e.preventDefault();
        void refetch();
      }}>
      {t('common.tryAgain')}
    </Button>
  }
  error={isError}>
  <ComponentContent />
</ErrorBoundary>
```

}
